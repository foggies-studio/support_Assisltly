# Support Assistly Bot

Telegram-бот поддержки на `Node.js + TypeScript + Telegraf v4`.

Бот разделяет два сценария:
- обычный пользователь отправляет обращение в поддержку;
- администратор получает тикет, видит мета-информацию и отвечает пользователю прямо из Telegram.

Проект не использует базу данных. Для MVP этого достаточно: баны хранятся в `data/bans.json`, а текущие временные состояния живут в памяти процесса.

## Возможности

- `/start` с поддержкой deep-link payload, например `/start from_secretary`
- пользовательская клавиатура:
  - `✍️ Написать в поддержку`
  - `ℹ️ Как описать проблему`
- прием одного следующего сообщения как обращения
- поддержка текста, фото, документов, видео, voice и других стандартных вложений через `copyMessage`
- отправка обращения в `ADMIN_CHAT_ID`
- отдельное мета-сообщение по каждому тикету:
  - `ticketId`
  - `userId`
  - `username`
  - `first_name`
  - дата и время
  - `startPayload`
- inline-кнопка `✉️ Ответить пользователю` под каждым тикетом
- отдельная админ-клавиатура для пользователей из `ADMIN_IDS`
- команды администратора:
  - `/reply <userId> <text>`
  - `/ban <userId>`
  - `/unban <userId>`
  - `/bans`
- rate limit на обращения: не чаще одного обращения в 20 секунд на пользователя
- баны через `data/bans.json`
- базовая устойчивость через `try/catch`, `bot.catch(...)` и логи в `console`

## Настройка `.env`

Создайте `.env` рядом с `.env.example`.

Пример:

```env
SUPPORT_BOT_TOKEN=123456789:YOUR_BOT_TOKEN
ADMIN_CHAT_ID=123456789
ADMIN_IDS=123456789,987654321
SUPPORT_NAME=Support Assistly
```

Описание переменных:

- `SUPPORT_BOT_TOKEN` - токен бота из BotFather
- `ADMIN_CHAT_ID` - чат, куда прилетают все обращения
- `ADMIN_IDS` - список Telegram ID администраторов через запятую
- `SUPPORT_NAME` - отображаемое имя поддержки, по умолчанию `Support Assistly`

## Как создать бота в BotFather

1. Откройте `@BotFather`.
2. Выполните `/newbot`.
3. Укажите имя и username бота.
4. Получите токен.
5. Запишите токен в `SUPPORT_BOT_TOKEN`.

## Как работает бот

### Пользователь

1. Пользователь отправляет `/start`.
2. Бот показывает приветствие и две кнопки:
   - `✍️ Написать в поддержку`
   - `ℹ️ Как описать проблему`
3. После нажатия `✍️ Написать в поддержку` бот ожидает одно следующее сообщение.
4. Следующее сообщение считается обращением и пересылается в `ADMIN_CHAT_ID`.
5. Пользователь получает ответ: `✅ Обращение принято. Мы ответим здесь.`

### Администратор

Если `userId` администратора указан в `ADMIN_IDS`, в `/start` он получает отдельную клавиатуру:

- `🛠 Панель ответов`
- `❌ Отмена ответа`
- `📋 Список банов`

Когда приходит тикет, бот отправляет:

- копию сообщения пользователя;
- мета-сообщение с информацией о тикете;
- кнопку `✉️ Ответить пользователю`.

Чтобы ответить:

1. Нажмите `✉️ Ответить пользователю` под нужным тикетом.
2. Отправьте следующее сообщение в чат с ботом.
3. Это сообщение будет отправлено пользователю.

Если нужно отменить режим ответа, нажмите `❌ Отмена ответа`.

## Команды администратора

Telegram автоматически показывает их в меню команд для пользователей из `ADMIN_IDS`.

- `/start` - показать панель
- `/reply <userId> <text>` - отправить текст пользователю вручную
- `/ban <userId>` - забанить пользователя
- `/unban <userId>` - разбанить пользователя
- `/bans` - вывести список забаненных пользователей

## Как узнать свой Telegram ID

Подходящие способы:

- написать `@userinfobot`
- написать `@RawDataBot`
- временно вывести `ctx.from?.id` в логах и написать боту

## Как настроить `ADMIN_CHAT_ID`

### Вариант 1: личный чат администратора

Используйте свой Telegram ID как `ADMIN_CHAT_ID`.

Подходит, если обращения должны приходить прямо вам в личку.

### Вариант 2: группа

Используйте ID группы вида `-100...` как `ADMIN_CHAT_ID`.

Важно:

- бот должен быть добавлен в группу;
- у него должны быть права на отправку сообщений;
- если вы хотите отвечать через inline-кнопку и админ-клавиатуру, администраторам все равно нужно отдельно открыть личный чат с ботом и выполнить `/start`.

Это означает, что после перезапуска процесса временные состояния сбрасываются. Для MVP это ожидаемое поведение.
