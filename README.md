# SupportBot (Support Assistly)

Отдельный Telegram support-бот (MVP) без БД на `Node.js + TypeScript + Telegraf v4`.

## 1) Создание бота в BotFather

1. Откройте Telegram и найдите `@BotFather`.
2. Выполните команду `/newbot`.
3. Укажите имя и username бота.
4. Скопируйте токен и сохраните в `.env` как `SUPPORT_BOT_TOKEN`.

## 2) Настройка `.env`

Создайте файл `.env` по примеру `.env.example`:

```env
SUPPORT_BOT_TOKEN=123456789:YOUR_BOT_TOKEN
ADMIN_CHAT_ID=123456789
ADMIN_IDS=123456789,987654321
SUPPORT_NAME=Support Assistly
```

Поля:
- `SUPPORT_BOT_TOKEN` - токен support-бота.
- `ADMIN_CHAT_ID` - чат, куда приходят обращения (личка админа или группа).
- `ADMIN_IDS` - Telegram ID админов через запятую.
- `SUPPORT_NAME` - опциональное название (по умолчанию `Support Assistly`).

## 3) Запуск

```bash
npm i
npm run dev
```

Продакшен-режим:

```bash
npm run build
npm start
```

## 4) Как узнать свой Telegram ID

Способы:
- написать `@userinfobot` и получить ID;
- временно добавить `console.log(ctx.from?.id)` в обработчик и отправить сообщение боту.

## 5) Как настроить `ADMIN_CHAT_ID`

### Личка админа
1. Напишите support-боту `/start`.
2. Узнайте свой ID (см. раздел выше).
3. Используйте этот ID как `ADMIN_CHAT_ID`.

### Группа
1. Добавьте support-бота в группу.
2. Дайте права на отправку сообщений.
3. Узнайте chat ID группы (обычно формат `-100...`) и укажите в `ADMIN_CHAT_ID`.

## 6) Админ-команды

- `/reply <userId> <text>` - ответить пользователю.
- `/ban <userId>` - забанить пользователя.
- `/unban <userId>` - разбанить пользователя.
- `/bans` - список забаненных пользователей.

## 7) Отдельная панель администратора

Если ваш `userId` есть в `ADMIN_IDS`, бот в `/start` показывает админ-клавиатуру:
- `🛠 Панель ответов`
- `❌ Отмена ответа`
- `📋 Список банов`

Как отвечать из админ-панели:
1. Получите новое обращение (бот отправит сообщение с мета-данными).
2. Нажмите кнопку `✉️ Ответить пользователю` под мета-сообщением.
3. Отправьте следующее сообщение в чат с ботом - оно уйдет пользователю.
4. Для отмены режима используйте `❌ Отмена ответа`.

Обычные пользователи продолжают видеть прежнюю клавиатуру:
- `✍️ Написать в поддержку`
- `ℹ️ Как описать проблему`

## 8) Что умеет бот

- `/start` с поддержкой deep-link payload (`/start from_secretary`).
- Отдельная панель для админов и обычная панель для пользователей.
- Приём одного следующего сообщения как обращения (текст или вложение).
- Пересылка обращения в `ADMIN_CHAT_ID` + отдельная мета-информация с `ticketId`, `userId`, `username`, `first_name`, временем и `startPayload`.
- Кнопка быстрого ответа админу (`✉️ Ответить пользователю`) под каждым тикетом.
- Ограничение частоты: не чаще 1 обращения в 20 секунд на пользователя.
- Баны в `data/bans.json`.
- Базовая устойчивость: `bot.catch(...)`, `try/catch`, консольные логи (`new ticket`, `reply`, `ban/unban`, `errors`).
